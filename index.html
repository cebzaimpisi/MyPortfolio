    1.Innovative Vehicle Tracking System

    User-Friendly Interface:



    The integration of a GSM, Bluetooth module HC-06 and a mobile phone as a user interface ensured a user-friendly experience. Users receive updates directly through their mobile devices, fostering easy interaction and real-time monitoring.

    Test the web app functionality through this link below:

    View on Link

    Project Overview

    The Innovative Vehicle Tracking System enhances vehicle security by tracking real-time location and speed using GPS, Bluetooth, Gas sensor, GSM, and LoRa technologies. The system provides seamless monitoring and geofencing capabilities, making it ideal for both personal vehicle security and fleet management solutions.

    Technologies Used

    • ATMEGA328P and ESP32 Microcontroller
    • Neo-8M GPS Module
    • HC-05 Module
    • Gas sensor Module (MQ2)
    • GSM Module for communication
    • LoRa Module for long-range, low-network areas
    • Programming Languages: C, C++ and JavaScript
    • Proteus for simulation
    • Google Maps API for location display
    • Atmel studio 7
    • Visual studio code
    • ThingSpeak (for live data sharing)

    System Circuit

    The system integrates GPS, GSM, and LoRa modules, allowing dual-layer communication for long-range tracking. In areas where GSM coverage is poor, the system seamlessly switches to LoRa to ensure uninterrupted location monitoring.

    Challenges and Solutions

    The main challenge was ensuring reliable communication in low-network areas. By integrating LoRa, a long-range, low-power communication system, the project achieved reliable data transmission over several kilometers, even in rural areas.

    Testing and Validation

    The system was tested across various environments to ensure reliability. It consistently provided location accuracy within 5 meters and had a GSM response time of under 10 seconds. LoRa extended the communication range, ensuring coverage in low-network areas.

    Impact and Future Improvements

    The system offers great potential in the automotive industry, particularly in fleet management and rural applications. Future improvements include integrating a mobile app for user-friendly tracking and adding advanced features like theft detection and emergency alerts.

    Project Duration

    This project took approximately **6 month** to complete, including the research, development,testing and presentation phases.

    3.Contactless Temperature Screening System

    A system for measuring temperature without contact.

    Project Overview

    The Contactless Temperature Screening System is designed to measure an individual’s body temperature without physical contact. This system provides an efficient solution for public places, ensuring safety during pandemic scenarios. The device uses an infrared sensor to capture temperature readings and displays them on an LCD screen.

    Technologies Used

    • ATMEGA328P Microcontroller
    • MLX90614 Infrared Temperature Sensor
    • LCD Display for output
    • Programming Languages: C, C++
    • Proteus for simulation

    System Circuit

    The system Circuit integrates the MLX90614 infrared sensor with an ESP32 microcontroller to process temperature readings and display them on an LCD. The diagram shows how each component interacts to ensure smooth operation.

    Challenges and Solutions

    One of the primary challenges was ensuring that the temperature readings were accurate at varying distances. By fine-tuning the sensor and calibrating the system, accurate readings were achieved within a range of 5-10 cm.

    Project Duration

    This project took approximately **3 month** to complete, including the research, development, testing and presentation phases.

    4.IoT-based Eskom Smart Meter System

    System Functionality Demo

    Project Description

    This smart meter integrates an ESP32 with Eskom’s circuit to enable wireless token entry, real-time energy monitoring, and remote validation. It ensures efficient energy usage and secure transactions using a dual-layer approach.

    Key Features

    • Wireless prepaid token entry with secure CRC-16 validation.
    • Real-time energy consumption logging with voltage, current, and kWh display.
    • Automatic power disconnection on overcurrent detection.
    • UART communication for remote token validation.
    • ThingSpeak integration for cloud-based energy data monitoring.
    • Mobile APP interface for wireless token recharge.

    Technologies Used

    Hardware: ESP32, ATmega328P, ZMPT101B (voltage sensor), Relay module, ACS712 (current sensor), DS3231 RTC, 4x4 keypad, 20×4 LCD (I2C).

    Software:Microchip Studio (Atmel Studio 7), ThingSpeak API, MATLAB IoT for data visualization.

    Programming languages: C, C++, JavaScript(with HTML and CSS).

    Implementation & Results: The system provides real-time energy tracking, ensures token security, and allows remote access for energy management..
