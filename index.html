1.Innovative Vehicle Tracking System

Project Overview

The Innovative Vehicle Tracking System enhances vehicle security by tracking real-time location using GPS, GSM, and LoRa technologies. The system provides seamless monitoring and geofencing capabilities, making it ideal for both personal vehicle security and fleet management solutions.

Technologies Used

  • ATMEGA328P and ESP32 Microcontroller
  • Neo-8M GPS Module
  • GSM Module for communication
  • LoRa Module for long-range, low-network areas
  • Programming Languages: C, C++ and JavaScript
  • Proteus for simulation
  • Google Maps API for location display

System Architecture

The system integrates GPS, GSM, and LoRa modules, allowing dual-layer communication for long-range tracking. In areas where GSM coverage is poor, the system seamlessly switches to LoRa to ensure uninterrupted location monitoring.

Challenges and Solutions

The main challenge was ensuring reliable communication in low-network areas. By integrating LoRa, a long-range, low-power communication system, the project achieved reliable data transmission over several kilometers, even in rural areas.

Testing and Validation

The system was tested across various environments to ensure reliability. It consistently provided location accuracy within 5 meters and had a GSM response time of under 10 seconds. LoRa extended the communication range, ensuring coverage in low-network areas.

Impact and Future Improvements

The system offers great potential in the automotive industry, particularly in fleet management and rural applications. Future improvements include integrating a mobile app for user-friendly tracking and adding advanced features like theft detection and emergency alerts.

2.Cooling Fan Speed Control

Project Overview

The Cooling Fan Speed Control App allows users to regulate the fan's speed based on ambient temperature or manual input through a mobile interface. This project aims to enhance energy efficiency by dynamically adjusting fan speed based on real-time temperature readings.

Key Features

  • Real-time temperature monitoring using a DHT11/DHT22 sensor.
  • Fan speed adjustment through PWM (Pulse Width Modulation).
  • User-friendly mobile app interface for manual control.
  • Automatic fan speed adjustment based on pre-set temperature thresholds.
  • Low power consumption mode to enhance energy efficiency.

Technologies Used

  • ESP32 Microcontroller
  • PWM for controlling fan speed
  • Mobile App Interface
  • Temperature Sensor (DHT11 or DHT22)
  • Programming Languages: C, C++ and JavaScript

Code Snippet


    // Function to control fan speed using PWM
    int fanPin = 5; // Pin connected to fan
    int tempPin = DHT11_PIN; // Pin connected to temperature sensor
    
    void setup() {
      pinMode(fanPin, OUTPUT);
      dht.begin();
    }
    
    void loop() {
      float temperature = dht.readTemperature();
      int pwmValue = map(temperature, 20, 40, 0, 255); // Adjust PWM based on temperature
      analogWrite(fanPin, pwmValue);
      delay(1000);
    }
3.Contactless Temperature Screening System

A system for measuring temperature without contact.

Final Product.

Project Overview

The Contactless Temperature Screening System is designed to measure an individual’s body temperature without physical contact. This system provides an efficient solution for public places, ensuring safety during pandemic scenarios. The device uses an infrared sensor to capture temperature readings and displays them on an LCD screen.

Technologies Used

  • ESP32 Microcontroller
  • MLX90614 Infrared Temperature Sensor
  • LCD Display for output
  • Programming Languages: C, C++
  • Proteus for simulation

System Architecture

The system architecture integrates the MLX90614 infrared sensor with an ESP32 microcontroller to process temperature readings and display them on an LCD. The diagram shows how each component interacts to ensure smooth operation.

Challenges and Solutions

One of the primary challenges was ensuring that the temperature readings were accurate at varying distances. By fine-tuning the sensor and calibrating the system, accurate readings were achieved within a range of 5-10 cm.

Project Duration

This project took approximately **3 month** to complete, including the research, development, and testing phases.
